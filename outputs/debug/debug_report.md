# PI-PHM 项目调试诊断报告

## Executive Summary

1. **P0 致命问题**: 模型目标语义不一致 - 模型设计意图预测位移增量，但实际训练目标是累计位移
2. **P0 致命问题**: 训练和测试评估不在同一量纲空间，导致无法正确监控模型性能  
3. **P0 致命问题**: 数据划分存在时间泄漏 - 验证集预测窗口跨越到测试集
4. **P0 致命问题**: PI-PHM性能(149.45mm MAE)比Persistence基线(2.82mm MAE)差50多倍
5. **P1 高优先级问题**: 主位移损失为0，模型完全未学习主任务
6. **P1 高优先级问题**: 物理约束损失项量级过大，淹没主任务损失

## Root Cause Ranking

### P0 致命问题

1. **目标语义不一致**
   - **现象**: 模型输出头注释明确表示"预测位移增量"，但数据集提供的是累计位移
   - **影响**: 模型学习错误的目标函数，导致预测结果无意义
   - **证据**: 
     - GNSS_12H原始数据范围: 0.16-716.48mm (累计位移)
     - 日变化量范围: -662.02 到 461.77mm
     - 模型应该预测日变化量，而不是累计值

2. **评估空间不一致**
   - **现象**: 训练时在归一化空间监控val_mae≈2.3，测试时在物理空间报告MAE=149.45mm
   - **影响**: 无法判断模型是否真正收敛，早停机制失效
   - **证据**: Trainer.validate() vs Evaluator.evaluate() 实现差异

3. **时间泄漏**
   - **现象**: 验证-测试间隙只有1天，但预测窗口为7天
   - **影响**: 验证集性能虚高，无法真实反映泛化能力
   - **证据**: val_end=2021-12-31, test_start=2022-01-01, forecast=7

4. **性能远差于基线**
   - **现象**: PI-PHM MAE=149.45mm vs Persistence基线 MAE=2.82mm
   - **影响**: 模型完全失效，不如最简单的基线
   - **证据**: 基线测试结果

### P1 高优先级问题

1. **主位移损失为0**
   - **现象**: 损失探针显示disp_loss=0.0，模型未学习主任务
   - **原因**: 目标定义错误导致损失计算异常

2. **物理约束损失主导**
   - **现象**: creep_loss=0.484, risk_loss=0.629 远大于主位移损失
   - **影响**: 模型优化方向偏离主任务

### P2 次要问题

1. **标签生成阈值可能不合理**
   - 风险标签基于固定阈值，可能需要根据数据分布动态校准

2. **manual Mamba实现稳定性**
   - 需要进一步验证manual Mamba的数值稳定性

## Evidence

### 目标语义审计结果
- GNSS_12H确认为累计位移（绝对值），单位mm
- 模型设计意图：预测位移增量
- 实际训练目标：预测累计位移
- **结论**: 语义不一致，P0问题

### 单位与量纲审计结果  
- 训练评估：归一化空间（无量纲）
- 测试评估：物理空间（mm）
- **结论**: 量纲不一致，P0问题

## Minimal Fix Plan

### 第一阶段修复 (必须先完成)

1. **修正目标定义** [P0]
   - 修改数据集__getitem__方法，将`y_disp_main`改为位移增量
   - 具体实现：`y_disp_main = current_gnss_values - last_input_gnss_value`
   - 同时修改辅助目标为增量形式

2. **统一评估空间** [P0]  
   - 修改Trainer.validate()方法，在反归一化空间计算MAE
   - 或者修改Evaluator.evaluate()，在归一化空间计算指标
   - 推荐：统一使用物理空间进行评估

3. **修复数据划分** [P0]
   - 增加验证-测试间隙至至少14天（> forecast_days）
   - 建议：val_end=2021-12-15, test_start=2022-01-01

### 第二阶段修复

4. **调整损失权重** [P1]
   - 确保主位移损失权重足够大
   - 暂时关闭物理约束损失，验证主任务能否正常学习

5. **验证小样本过拟合** [P1]
   - 在修正目标后，运行小样本过拟合测试
   - 确认模型架构和实现正确性

### 第三阶段优化

6. **校准风险标签阈值** [P2]
   - 基于训练集数据分布动态计算阈值
   - 避免固定阈值导致的类别不平衡

7. **验证manual Mamba** [P2]
   - 对比Identity/GRU/Mamba三种实现的性能

## Conclusion

当前PI-PHM项目存在**多个P0级别的致命问题**，主要根源在于**目标语义定义错误**。在修复这些问题之前，任何模型架构调优、超参数调整或复杂功能添加都是徒劳的。

**建议优先级**: 目标修正 > 评估统一 > 数据划分修复 > 损失权重调整

修复后，预期性能应至少达到Persistence基线水平(MAE < 3mm)，然后才能考虑进一步优化。
